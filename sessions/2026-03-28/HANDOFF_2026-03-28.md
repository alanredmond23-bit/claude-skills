## KEY ARCHITECTURE DECISIONS MADE THIS SESSION

1. Prompt Refiner uses UserPromptSubmit hook — not a wrapper script. Fires natively in Claude Code CLI.
2. Multi-Model Orchestrator uses `run_in_executor` for Gemini and Claude SDK (not natively async) rather than switching to httpx.
3. Grok client uses xAI's OpenAI-compatible endpoint — same client library, different base_url. No separate SDK needed.
4. PDF uses ReportLab not WeasyPrint — better M1 Mac compatibility.
5. Consensus engine uses TF-IDF + cosine similarity (sklearn) — not LLM-based scoring. Keeps it offline and fast.
6. All runs stored as directory trees with individual response .txt files — easy to `cat` or grep without parsing JSON.
